# GroupMe Exporter Architecture

## Overview

This system continuously exports messages from a GroupMe chat into a local SQLite database and periodically snapshots the database to network storage.

Two independent components:

| Component | Mechanism | Purpose |
|---|---|---|
| Ingestion daemon | systemd | Polls GroupMe API, writes to SQLite |
| Snapshot job | root cron | Backs up SQLite to network storage every 30 min |

These are intentionally decoupled — ingestion continues even if network storage is unavailable.

---

## Host Layout

Default application root: `$GROUPME_INSTALL_DIR` (typically `/opt/groupme`)

| File / Dir | Purpose |
|---|---|
| `groupme_ingest.py` | Main ingestion daemon |
| `progress.py` | Ingestion checkpoint tracking |
| `verify_coverage.py` | Message completeness validation |
| `groupme_schema.sql` | SQLite schema |
| `groupme.sqlite` | Primary message database (WAL mode) |
| `snapshot.sh` | Database snapshot script |
| `snapshot.lock` | Prevents concurrent snapshot runs |
| `snapshot.err` | Snapshot error log |
| `tmp/` | Temporary snapshot workspace |
| `venv/` | Python virtual environment |

## Secrets and Configuration

Secrets live outside the repository at `/etc/groupme.env` (see `.env.example`):

```
GROUPME_TOKEN=<token>
GROUPME_GROUP_ID=<group_id>
GROUPME_INSTALL_DIR=/opt/groupme
```

`GROUPME_TOKEN` — GroupMe developer API token (from dev.groupme.com).

`GROUPME_GROUP_ID` — numeric ID of the target group. To find it, call the
GroupMe API: `GET https://api.groupme.com/v3/groups?token=YOUR_TOKEN` and
look for the `id` field of the group you want.

The systemd service loads this file via `EnvironmentFile=/etc/groupme.env`.
Secrets are never stored in the repository.

Recommended permissions:
```bash
sudo chmod 600 /etc/groupme.env
sudo chown root:root /etc/groupme.env
```

## Deployment

The `scripts/install.sh` script handles deployment. It reads `GROUPME_INSTALL_DIR`
from `/etc/groupme.env`, copies source files flat into that directory, creates a
Python venv, and generates the real service file from `systemd/groupme-daemon.service`
by substituting `%%INSTALL_DIR%%` with the configured path.

To deploy or update:
```bash
sudo bash scripts/install.sh [/path/to/env/file]
```

## Ingestion Daemon

Service file: `/etc/systemd/system/groupme-daemon.service` (generated by install.sh)

Command (with defaults expanded):
```
$GROUPME_INSTALL_DIR/venv/bin/python -u $GROUPME_INSTALL_DIR/groupme_ingest.py \
  --daemon --interval 30 --head-pages 6 --reconcile-head 6 --verbose
```

### Startup sequence

1. **Backfill** — walks historical messages backward via `before_id`, resuming from the saved checkpoint
2. **Topoff** — fetches newest pages to catch any messages missed during backfill
3. **Reconcile** — rescans newest pages to pick up like/reaction edits and removals
4. **Daemon loop** — repeats topoff + reconcile every 30 seconds

### Checkpoint

Progress is saved to the `ingestion_progress` table in SQLite after each page. On restart, backfill resumes from the last saved `before_id`.

## Snapshot System

Script: `$GROUPME_INSTALL_DIR/snapshot.sh`

Cron entry (root):
```
*/30 * * * * /opt/groupme/snapshot.sh >> /var/log/groupme_snapshot.log 2>&1
```

### What the script does

1. Checks that the snapshot destination is mounted and writable — skips gracefully if not
2. Acquires a lock file to prevent concurrent runs
3. Creates a safe offline copy of the SQLite database via `.backup`
4. Copies the snapshot to the destination
5. Updates `groupme_latest.sqlite` (stable filename for downstream consumers)
6. Prunes old snapshots, keeping the 3 most recent

### Snapshot storage

Snapshots are written to `GROUPME_SNAPSHOT_DEST` (default: `/mnt/groupme_snapshots`).

A typical setup uses an SMB share mounted via `/etc/fstab`. See `docs/fstab.example`
for a reference mount configuration using a bind mount pattern:

```
//your-smb-server/share          → /mnt/smb_root              (CIFS mount)
/mnt/smb_root/groupme_snapshots  → /mnt/groupme_snapshots     (bind mount)
```

Snapshot files:
```
groupme_YYYYMMDD-HHMMSS.sqlite   # timestamped archive
groupme_latest.sqlite             # stable symlink for consumers
```

## Runtime Flow

```
GroupMe API
    │
    ▼
groupme_ingest.py  (systemd daemon, polls every 30s)
    │
    ▼
$GROUPME_INSTALL_DIR/groupme.sqlite  (SQLite, WAL mode)
    │
    ▼
snapshot.sh  (root cron, every 30m)
    │
    ▼
$GROUPME_SNAPSHOT_DEST  (network storage)
```

## Monitoring

```bash
# Daemon status
sudo systemctl status groupme-daemon

# Live daemon logs
sudo journalctl -u groupme-daemon -f

# Snapshot logs
tail -f /var/log/groupme_snapshot.log

# Message count and ingestion progress
python $GROUPME_INSTALL_DIR/progress.py

# Completeness check against GroupMe API
python $GROUPME_INSTALL_DIR/verify_coverage.py
```
